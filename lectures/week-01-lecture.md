# Week 01 Lecture Notes: How the Browser Parses, Paints, and Executes

## Weekly focus
The Browser as a Platform

## Why this matters
This week's material helps students move from isolated syntax toward purposeful client-side design decisions. The lecture emphasizes examples that can be inspected in the browser and rebuilt in small pieces.

## Learning targets
- Explain the core concept in plain language
- Build one focused example from scratch
- Debug mistakes with browser tools
- Connect this week's ideas to the running course projects

## Lecture flow
1. Warm-up concept check
2. Demonstration of a focused example
3. Guided breakdown of the code
4. Common bug patterns
5. Independent practice prompt
6. Bridge to lab, assignment, or quiz

## Instructor notes
Use short code examples, narrate tradeoffs, and revisit accessibility in every demo. Avoid introducing abstractions before students can inspect the DOM, state, or network behavior involved.

## Exit ticket
Ask students to explain one idea from the week using the sentence frame: 'This matters in the browser because...'
