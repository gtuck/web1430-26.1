Chapter 2: HTML, CSS, and Progressive Enhancement

The chapter revisits semantic structure, responsive layout, and layered enhancement so students understand why client-side scripting should strengthen content rather than replace it.

What this chapter is really about

The chapter revisits semantic structure, responsive layout, and layered enhancement so students understand why client-side scripting should strengthen content rather than replace it. Rather than treating the browser like a magic box, this chapter asks students to name the moving parts, decide where responsibility belongs, and make design decisions visible in code.

Key ideas

The web is layered. Content comes first, presentation follows, behavior enhances, and data flows across boundaries. Good client-side code respects those layers instead of blurring them into one large script.

Mental model

Each page is a conversation between structure, style, state, and time. Structure gives the page meaning. Style gives it shape. State tracks what the user has done and what the application knows. Time matters because users click, type, navigate, and wait.

Working habits

Students are encouraged to experiment in small loops: change one thing, observe the result, document the reason, and commit progress in Git. The goal is not simply to make a page work; the goal is to understand why it works.

Common mistakes

Many beginners overreach too early. They add libraries before they understand the browser, hide logic in anonymous handlers, or skip naming conventions that would make the code readable later. This chapter pushes against that pattern.

Accessibility and UX note

Every technical choice has an experience consequence. Heading structure affects screen-reader navigation. Button labeling affects confidence. Error messaging affects recovery. A client-side developer is always shaping user experience, even when writing simple scripts.

Practice prompt

Open one of your existing pages or a provided course starter. Identify one part of the interface that depends on html, css, and progressive enhancement and explain what the browser needs in order to make that feature understandable and usable.

Reflection

What did you assume the browser was doing automatically that actually depends on deliberate authoring? Where can you make your intent more visible in HTML, CSS, or JavaScript?
